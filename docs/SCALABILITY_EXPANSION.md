# Scalability and Expansion

## Nigeria Expansion

Prepare for:

- All Nigerian states.
- LGAs.
- Communities.
- Multiple Nigerian languages.
- Regional vendor verification workflows.

## African Expansion

Future multi-country support requires:

- Country model.
- Currency support.
- Time zone support.
- Local payment providers.
- Local address formats.
- Local compliance review.

## Database Indexing

Prioritize indexes for:

- Vendor status and location.
- Vendor state/LGA/community.
- Menu category.
- Order customer/vendor/status.
- Payment provider reference.
- Review vendor/status.
- Audit/security logs.

## Caching

Candidates:

- Food categories.
- Public vendor profiles.
- Search filters.
- Localization bundles.

## Search Scaling

Start with PostgreSQL filters. Add dedicated search later if needed:

- Meilisearch.
- OpenSearch.
- Postgres full-text search.

## Maps Cost Control

- Lazy-load map scripts.
- Cache geocoding where allowed.
- Prefer directions links for simple navigation.
- Restrict API keys by domain/app.

## Payment Fallback

Keep provider abstraction for:

- Paystack.
- Flutterwave.
- Moniepoint.
- Opay.
- PalmPay.
- Bank transfer.
- Wallet.
- Card.

## Moderation Scaling

- Regional admin queues.
- Risk scoring with human review.
- Vendor verification checklists.
- Audit trails for every decision.
