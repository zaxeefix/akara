# Support Operations

## Support Channels

- Help Center.
- Contact support form placeholder.
- GitHub issues for contributor-facing bugs.
- Future email or ticketing tool.

## Support Categories

- Customer order issue.
- Payment issue.
- Vendor onboarding issue.
- Vendor approval issue.
- Review/report issue.
- Security concern.
- Technical bug.

## Handling Complaints

1. Capture user role and contact.
2. Capture order/vendor/reference ID where available.
3. Categorize severity.
4. Check audit, security, order, and payment logs.
5. Escalate safety and payment issues.
6. Document resolution.

## Response Priorities

- Critical: payment/security/safety.
- High: order blocking or vendor approval issue.
- Medium: profile, menu, translation, or review issue.
- Low: documentation or cosmetic issue.
