# Information Architecture Refinement

## Customer Side

### Primary Navigation

- Home
- Search vendors
- Nearby vendors
- Food categories
- Orders
- Favorites
- Notifications
- Profile

### Secondary Customer Areas

- Vendor details
- Cart
- Checkout
- Order tracking
- Reviews
- Language settings
- Theme settings
- Saved addresses
- Payment methods

### Customer Navigation Model

Desktop:

- Header navigation with logo, search, location, categories, cart, notifications, and profile menu.
- Profile menu contains orders, favorites, language settings, theme settings, and logout.

Mobile:

- Sticky top search and location bar.
- Bottom navigation: Home, Search, Nearby, Orders, Profile.
- Cart appears as floating shortcut when items exist.
- Language settings live in Profile and onboarding.

## Vendor Side

### Primary Navigation

- Vendor onboarding
- Vendor dashboard
- Business profile
- Menu management
- Orders
- Sales analytics
- Earnings
- Reviews
- Notifications
- Verification
- Settings

### Vendor Navigation Model

Desktop:

- Persistent sidebar with grouped navigation.
- Top bar with order alert, notification bell, language, and profile.

Mobile:

- Bottom navigation: Dashboard, Orders, Menu, Earnings, More.
- More contains Business Profile, Reviews, Verification, Notifications, Settings.

### Vendor Onboarding Structure

1. Account setup
2. Business identity
3. Contact information
4. Location and map pin
5. Opening hours
6. Photos and description
7. Review and submit
8. Approval status

## Admin Side

### Primary Navigation

- Admin dashboard
- Vendor approval
- Customer management
- Vendor management
- Orders
- Payments
- Fraud detection
- Reports
- Ads management
- Subscription plans
- Security logs
- Localization management
- Settings

### Admin Navigation Model

Desktop:

- Persistent sidebar.
- Top bar with global admin search, alerts, and profile.
- Dashboard surfaces high-priority queues.

Tablet:

- Collapsible sidebar.
- Tables remain readable through responsive columns and detail panels.

Mobile:

- Admin should remain usable for urgent actions.
- Use drawer navigation and stacked cards.
- Avoid requiring complex bulk operations on small screens.

- Language switcher

- Security settings
- Help/support

## Content Hierarchy

### Customer Priority

1. Find food or vendors.
2. Confirm trust and location.
3. Place order.
4. Track order.
5. Review and reorder.

### Vendor Priority

1. Know current order demand.
2. Keep menu accurate.
3. Manage order status.
4. Track earnings.
5. Maintain trust and verification.

### Admin Priority

1. Approve and moderate vendors.
2. Detect fraud and suspicious activity.
3. Monitor orders and payments.
4. Manage users and platform settings.
5. Review reports and growth metrics.
